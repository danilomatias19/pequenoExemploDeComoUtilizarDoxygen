		1.1 - Definição
			"Doxygen é um gerador de documentação para C++, C, Java, Objective-C, Python, IDL (CORBA e variações da Microsoft), Fortran, VHDL, PHP, C#, e, até certo ponto, D.
			Este programa roda em sistemas Unix-like, Mac OS X, e no Windows." (Wikipedia, Doxygen. Disponível em:<http://pt.wikipedia.org/wiki/Doxygen>.Acesso em: 18 de maio de 2012).
			"Doxygen é uma ferramenta para geração de documentação e referências de códigos. A documentação é escrita utilizando código fonte e isto é realmente (incrivelmente) fácil." (Wikipedia)
			"Da mesma forma que o Javadoc, o Doxygen extrai a documentação dos comentários do arquivo fonte. Além da syntax do Javadoc, o Doxygen suporta as marcações de documentação usadas no Qt toolkit e sua saída pode ser gerada em HTML bem como em CHM, RTF, PDF, LaTeX, PostScript ou man pages." (Wikipedia)

		3.1 - Pequeno exemplo
			Para não perdermos tempo codificando arquivos .php acesse:
			http://10.1.1.17/7/teste/phps/2_php/2_php.txt
			e
			http://10.1.1.17/7/teste/phps/2/confsDoxygen/Doxyfileconf2php.txt
			O comando da instação já baixa o exemplo e a configuração do doxygen
			wget -c http://10.1.1.17/7/teste/phps/2_php/2_php.txt;wget -c http://10.1.1.17/7/teste/phps/2/confsDoxygen/Doxyfileconf2php.txt


			3.1.1 - Agora no terminal (Aplicativos > Acessórios > Terminal) da minha vm (No seu computador) dou um:
				doxywizard
					Após ter aberto o doxywizard
					3.1.1.1 - Carrego o arquivo de configuração Doxyfileconf2php.txt
						Para isso na tela do doxygen aberta, vou em
							File > open
								Seleciono o documento ~/Documentos/doxygendm1/Doxyfileconf2php.txt
									No campo Source Code Directory, clico no botão select,  e o
									faço apontar para o caminho onde está meu código fonte, nesse caso no diretório, ~/Documentos/doxygendm1/source
									Já no campo, Destination Directory, clico no  botão select,  e
									faço apontar para o caminho onde será gerada a documentação com o doxygen, nesse caso no diretório, ~/Documentos/doxygendm1/OUTPUT_DIRECTORY
					3.1.1.2 - Para certificar-me de que o graphviz realmente encontra-se em /usr/lib/graphviz
						Abro o terminal (Aplicativos > Acessórios > Terminal)  e digito:
							whereis graphviz
						Se aparecer:
							graphviz: /usr/lib/graphviz ...
						Está tudo certo, se não copie o que estiver aparecendo no lugar de:
							/usr/lib/graphviz
						Abra a aba
							 Experts do doxywizard
								Após, na árvore da esquerda, procure pelo nó Dot, se necessário mova a barra de rolagem de maneira a visualizar toda a árvore, após encontrar   o nó Dot, clique nele
									No formulário que apareceu, procure pelo campo (Se necessário, mova a barra de rolagem da direita, para baixo) :
										DOT_PATH
											Substitua o /usr/lib/graphviz, pelo conteúdo copiado, ou então cole em cima de  /usr/lib/graphviz
						Se aparecer graphviz: /usr/lib/graphviz ..., também vá na aba Experts para conferir se o campo  DOT_PATH, está realmente
						preenchido com /usr/lib/graphviz, se estiver ok, está tudo certo.

					3.1.1.3 - Abra a aba Run do doxywizard
						Clique no botão Show Configuration
							Após clique no botão Run Doxygen
								Após clique no botão Show HTML Output
						E confira o resultado da documentação gerada pelo doxygen apartir do arquivo:
						~/Documentos/doxygendm1/source/2.php
		3.2-	Não gostou dos arquivos ?
			Apague tudo de uma vez digitando no terminal:
				rm -rfv ~/Documentos/doxygendm1/
					Para desinstalar o doxygen:
						sudo apt-get --purge remove graphviz libgraphviz4 doxygen doxygen-gui
